Testing Strategy

A comprehensive testing strategy must be implemented to ensure the quality and correct functioning of the system.

  • Unit Tests: To verify the correct functioning of individual code components.
  • Integration Tests: To ensure that different system modules interact correctly with each other.
  • System Tests: To validate the complete system against functional and non-functional requirements.
  • User Acceptance Tests (UAT): Performed by end-users (kindergarten staff) to ensure that the system meets their expectations and needs.

Test-Driven Development (TDD)

Consider adopting TDD, where tests are written before production code. This helps ensure that every functionality is covered by tests and improves code design.

Testing Tools

Selection of appropriate tools for each type of test.

  • Unit/Integration: JUnit (Java), Pytest (Python), Jest (JavaScript), PHPUnit (PHP).
  • API (End-to-End): Postman, Insomnia, Hurl (as seen in the current project).
  • UI/E2E: Selenium, Cypress, Playwright.

Examples of Tests with Hurl (for API)

Following the example of the current project, .hurl files can be defined to test API endpoints.

Example: Test the creation of a new student

# 001_create_student.hurl
POST http://localhost:8080/api/students
Content-Type: application/json

Testing Environment

Configuration of an isolated testing environment that replicates the production environment as closely as possible, but with test data.

Considerations:

  • Separate test database.
  • Anonymized or generated test data.
  • Automation of deployment to the testing environment.
