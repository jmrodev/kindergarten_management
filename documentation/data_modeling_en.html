Detailed Use Cases

Extension of the use cases identified in the business analysis phase, including alternative flows and exceptions.

Example:

  • Use Case: Update Student Address.
  • Actor: Secretary.
  • Precondition: The student exists in the system.
  • Main Flow:
    1. Secretary searches for the student by name or ID.
    2. System displays student data, including current address.
    3. Secretary edits the address fields.
    4. Secretary saves the changes.
    5. System validates the data and updates the address.
    6. System confirms successful update.
  • Alternative Flow: Invalid address data.
  • Postcondition: The student's address has been updated in the database.

Example:

  • Use Case: Register New Student.
  • Actor: Secretary, Kindergarten Administrator.
  • Precondition: The user has logged in and has permissions to register students.
  • Main Flow:
    1. User navigates to the student registration section.
    2. System presents the student registration form.
    3. User enters mandatory student data (first name, middle name, nickname, paternal surname, maternal surname, birth date, classroom, shift).
    4. User enters information for at least one parent/guardian (first name, paternal surname, maternal surname, relationship, phone, email, authorized for pickup, authorized for change).
    5. User enters the student's primary address.
    6. User saves the record.
    7. System validates the entered data.
    8. System creates the new student, parent/guardian, and address record in the database.
    9. System confirms successful registration and displays the student in the list.
  • Alternative Flow: Missing or invalid mandatory data.
    1. System detects missing or invalid data.
    2. System displays error messages indicating fields to correct.
    3. User corrects the data and retries saving.
  • Postcondition: A new student, with their associated parent/guardian and address, has been successfully registered in the system.

Data Dictionary

Detailed documentation of each database field, including data type, length, constraints, description, and whether it is mandatory. This section details the entities and their main attributes, based on the scopes defined in the Business Analysis.

Entity: Student

  • id: BIGINT, Primary Key. Unique identifier for the student.
  • first_name: TEXT, Required. Student's first name.
  • middle_name_optional: TEXT, Optional. Student's middle name.
  • paternal_surname: TEXT, Required. Student's paternal surname.
  • maternal_surname: TEXT, Required. Student's maternal surname.
  • preferred_surname: TEXT, Optional. Student's preferred surname.
  • nickname_optional: TEXT, Optional. Student's nickname.
  • birth_date: DATE, Required. Student's date of birth.
  • address_id: BIGINT, Foreign Key. Reference to Address Entity. Student's primary address.
  • emergency_contact_id: BIGINT, Foreign Key. Reference to Emergency Contact Entity. Emergency contact information.
  • classroom_id: BIGINT, Foreign Key. Reference to Classroom Entity. Classroom the student belongs to.
  • shift: TEXT, Required. Student's shift (e.g., 'Morning', 'Afternoon', 'Full Day').

Entity: Guardian

  • id: BIGINT, Primary Key. Unique identifier for the guardian.
  • first_name: TEXT, Required. Guardian's first name.
  • middle_name_optional: TEXT, Optional. Guardian's middle name.
  • paternal_surname: TEXT, Required. Guardian's paternal surname.
  • maternal_surname: TEXT, Required. Guardian's maternal surname.
  • preferred_surname: TEXT, Optional. Guardian's preferred surname.
  • relationship: TEXT, Required. Relationship with the student (e.g., 'Father', 'Mother', 'Tutor').
  • address_id: BIGINT, Foreign Key. Reference to Address Entity. Guardian's address.
  • phone: TEXT, Required. Contact phone number.
  • email_optional: TEXT, Optional. Contact email.
  • authorized_pickup: BOOLEAN, Required. Indicates if authorized to pick up the student.
  • authorized_change: BOOLEAN, Required. Indicates if authorized to make changes for the student.

Entity: Staff

  • id: BIGINT, Primary Key. Unique identifier for the staff member.
  • first_name: TEXT, Required. Staff member's first name.
  • middle_name_optional: TEXT, Optional. Staff member's middle name.
  • paternal_surname: TEXT, Required. Staff member's paternal surname.
  • maternal_surname: TEXT, Required. Staff member's maternal surname.
  • preferred_surname: TEXT, Optional. Staff member's preferred surname.
  • address_id: BIGINT, Foreign Key. Reference to Address Entity. Staff member's address.
  • phone: TEXT, Required. Contact phone number.
  • email_optional: TEXT, Optional. Contact email.
  • classroom_id: BIGINT, Foreign Key. Reference to Classroom Entity. Main classroom assigned to the staff member.
  • role_id: BIGINT, Foreign Key. Reference to Role Entity. Staff member's role.

Entity: Address

  • id: BIGINT, Primary Key. Unique identifier for the address.
  • street: TEXT, Required. Street name.
  • number: TEXT, Required. House or building number.
  • city: TEXT, Required. City.
  • state: TEXT, Required. State/Province.
  • postal_code_optional: TEXT, Optional. Postal code.

Entity: Emergency Contact

  • id: BIGINT, Primary Key. Unique identifier for the emergency contact.
  • full_name: TEXT, Required. Full name of the emergency contact.
  • relationship: TEXT, Required. Relationship with the student.
  • phone: TEXT, Required. Emergency contact phone number.

Entity: Classroom

  • id: BIGINT, Primary Key. Unique identifier for the classroom.
  • name: TEXT, Required. Name of the classroom (e.g., 'Red Room', 'Green Room').
  • capacity: INT, Required. Maximum student capacity.

Entity: Attendance

  • id: BIGINT, Primary Key. Unique identifier for the attendance record.
  • student_id: BIGINT, Foreign Key. Reference to Student Entity. Student to whom the attendance corresponds.
  • date: DATE, Required. Date of attendance.
  • status: TEXT, Required. Attendance status (e.g., 'Present', 'Absent', 'Late', 'Justified').
  • leave_type_optional: TEXT, Optional. Type of leave if applicable (e.g., 'Illness', 'Personal').
  • classroom_id: BIGINT, Foreign Key. Reference to Classroom Entity. Classroom where attendance was taken.

Entity: Calendar

  • id: BIGINT, Primary Key. Unique identifier for the calendar event.
  • date: DATE, Required. Date of the event.
  • description: TEXT, Required. Description of the event.
  • event_type: TEXT, Required. Type of event (e.g., 'Holiday', 'Leave', 'Kindergarten Activity', 'Classroom Activity', 'Community Activity', 'Country Activity').
  • classroom_id: BIGINT, Foreign Key. Reference to Classroom Entity. Classroom associated with the event (if it's a classroom activity).

Entity: Activity

  • id: BIGINT, Primary Key. Unique identifier for the activity.
  • name: TEXT, Required. Name of the activity (e.g., 'Music', 'Body Expression').
  • description_optional: TEXT, Optional. Description of the activity.
  • schedule_optional: TEXT, Optional. Schedule of the activity.
  • teacher_id: BIGINT, Foreign Key. Reference to Staff Entity. Teacher in charge of the activity.

Entity: Conversation

  • id: BIGINT, Primary Key. Unique identifier for the conversation.
  • classroom_id: BIGINT, Foreign Key. Reference to Classroom Entity. Classroom associated with the conversation (if applicable).
  • creation_date: TIMESTAMP, Required. Date and time of conversation creation.

Entity: Access Level

  • id: BIGINT, Primary Key. Unique identifier for the access level.
  • access_name: TEXT, Required. Name of the access level.
  • description: TEXT, Optional. Description of the access level.

Entity: Role

  • id: BIGINT, Primary Key. Unique identifier for the role.
  • role_name: TEXT, Required. Name of the role.
  • access_level_id: BIGINT, Foreign Key. Reference to Access Level Entity. Access level associated with this role.

Entity: Conversation Guardian

  • conversation_id: BIGINT, Primary Key, Foreign Key. Reference to Conversation Entity.
  • guardian_id: BIGINT, Primary Key, Foreign Key. Reference to Guardian Entity.

Entity: Conversation Staff

  • conversation_id: BIGINT, Primary Key, Foreign Key. Reference to Conversation Entity.
  • staff_id: BIGINT, Primary Key, Foreign Key. Reference to Staff Entity.

Entity: Guardian Message

  • id: BIGINT, Primary Key. Unique identifier for the guardian message.
  • conversation_id: BIGINT, Foreign Key. Reference to Conversation Entity. Conversation to which the message belongs.
  • sender_guardian_id: BIGINT, Foreign Key. Reference to Guardian Entity. ID of the guardian who sent the message.
  • content: TEXT, Required. Content of the message.
  • send_date: TIMESTAMP, Required. Date and time the message was sent.
  • read: BOOLEAN, Required. Indicates if the message has been read.

Entity: Staff Message

  • id: BIGINT, Primary Key. Unique identifier for the staff message.
  • conversation_id: BIGINT, Foreign Key. Reference to Conversation Entity. Conversation to which the message belongs.
  • sender_staff_id: BIGINT, Foreign Key. Reference to Staff Entity. ID of the staff member who sent the message.
  • content: TEXT, Required. Content of the message.
  • send_date: TIMESTAMP, Required. Date and time the message was sent.
  • read: BOOLEAN, Required. Indicates if the message has been read.
