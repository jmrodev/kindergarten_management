Work Methodology

An agile methodology (Scrum or Kanban) is recommended for the development of this project, allowing flexibility and adaptation to the changing requirements of the kindergarten.

  • Roles: Product Owner, Scrum Master, Development Team.
  • Events: Sprints, Sprint Planning, Daily Scrum, Sprint Review, Sprint Retrospective.
  • Artifacts: Product Backlog, Sprint Backlog, Increment.

Planning

Gantt / PERT Chart

Use Gantt or PERT charts to visualize project tasks, dependencies, and deadlines. This will help maintain clear progress tracking.

Example of key tasks:

  • Initial requirements gathering
  • Database design
  • Development of the student management module
  • Implementation of user authentication
  • Integration testing
  • Initial deployment

Version Control

It is essential to use a version control system like Git. A branch-based workflow (Git Flow or GitHub Flow) is recommended to efficiently manage the source code.

Key points:

  • `main` or `master` branch for stable production code.
  • `develop` branches for integrating new functionalities.
  • `feature` branches for developing individual features.
  • `hotfix` branches for urgent production fixes.
  • Clear and descriptive commits.
