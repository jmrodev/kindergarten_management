Casos de Uso Detallados

Ampliación de los casos de uso identificados en la fase de análisis de negocio, incluyendo flujos alternativos y excepciones.

Ejemplo:

  • Caso de Uso: Actualizar Dirección de Alumno.
  • Actor: Secretaria/o.
  • Precondición: El alumno existe en el sistema.
  • Flujo Principal:
    1. Secretaria/o busca al alumno por nombre o ID.
    2. Sistema muestra los datos del alumno, incluyendo la dirección actual.
    3. Secretaria/o edita los campos de la dirección.
    4. Secretaria/o guarda los cambios.
    5. Sistema valida los datos y actualiza la dirección.
    6. Sistema confirma la actualización exitosa.
  • Flujo Alternativo: Datos de dirección inválidos.
  • Postcondición: La dirección del alumno ha sido actualizada en la base de datos.

Ejemplo:

  • Caso de Uso: Registrar Nuevo Alumno.
  • Actor: Secretaria/o, Administrador del Jardín.
  • Precondición: El usuario ha iniciado sesión y tiene permisos para registrar alumnos.
  • Flujo Principal:
    1. Usuario navega a la sección de registro de alumnos.
    2. Sistema presenta el formulario de registro de alumnos.
    3. Usuario ingresa los datos obligatorios del alumno (nombre, segundo nombre, alias, apellido materno, apellido paterno, fecha de nacimiento, sala, turno).
    4. Usuario ingresa la información de al menos un padre/tutor (nombre, apellido materno, apellido paterno, relación, teléfono, email, autorizado para retiro, autorizado para cambiado).
    5. Usuario ingresa la dirección principal del alumno.
    6. Usuario guarda el registro.
    7. Sistema valida los datos ingresados.
    8. Sistema crea el nuevo registro de alumno, padre/tutor y dirección en la base de datos.
    9. Sistema confirma el registro exitoso y muestra el alumno en el listado.
  • Flujo Alternativo: Datos obligatorios faltantes o inválidos.
    1. Sistema detecta datos faltantes o inválidos.
    2. Sistema muestra mensajes de error indicando los campos a corregir.
    3. Usuario corrige los datos y reintenta guardar.
  • Postcondición: Un nuevo alumno, con su padre/tutor asociado y dirección, ha sido registrado exitosamente en el sistema.

Diccionario de Datos

Documentación detallada de cada campo de la base de datos, incluyendo tipo de dato, longitud, restricciones, descripción y si es obligatorio. Esta sección detalla las entidades y sus atributos principales, basándose en los alcances definidos en el Análisis de Negocio.

Entidad: Alumno

  • nombre: VARCHAR(50), Obligatorio. Primer nombre del alumno.
  • segundo_nombre: VARCHAR(50), Opcional. Segundo nombre del alumno.
  • tercer_nombre: VARCHAR(50), Opcional. Tercer nombre del alumno.
  • alias: VARCHAR(50), Opcional. Apodo o nombre corto del alumno.
  • apellido_materno: VARCHAR(50), Obligatorio. Apellido materno del alumno.
  • apellido_paterno: VARCHAR(50), Obligatorio. Apellido paterno del alumno.
  • fecha_nacimiento: DATE, Obligatorio. Fecha de nacimiento del alumno.
  • direccion: Referencia a Entidad Dirección, Obligatorio. Dirección principal del alumno.
  • contacto_emergencia: Referencia a Entidad ContactoEmergencia, Obligatorio. Información de contacto para emergencias.
  • sala_id: Referencia a Entidad Sala, Obligatorio. Sala a la que pertenece el alumno.
  • turno: ENUM('Mañana', 'Tarde', 'Completo'), Obligatorio. Turno del alumno.

Entidad: Padre/Tutor

  • nombre: VARCHAR(50), Obligatorio. Primer nombre del padre/tutor.
  • segundo_nombre: VARCHAR(50), Opcional. Segundo nombre del padre/tutor.
  • tercer_nombre: VARCHAR(50), Opcional. Tercer nombre del padre/tutor.
  • apellido_materno: VARCHAR(50), Obligatorio. Apellido materno del padre/tutor.
  • apellido_paterno: VARCHAR(50), Obligatorio. Apellido paterno del padre/tutor.
  • relacion: VARCHAR(50), Obligatorio. Relación con el alumno (ej. 'Padre', 'Madre', 'Tutor').
  • direccion: Referencia a Entidad Dirección, Obligatorio. Dirección del padre/tutor.
  • telefono: VARCHAR(20), Obligatorio. Número de teléfono de contacto.
  • email: VARCHAR(100), Opcional. Correo electrónico de contacto.
  • autorizado_retiro: BOOLEAN, Obligatorio. Indica si está autorizado para retirar al alumno.
  • autorizado_cambiado: BOOLEAN, Obligatorio. Indica si está autorizado para cambiar al alumno.

Entidad: Personal

  • nombre: VARCHAR(50), Obligatorio. Primer nombre del personal.
  • segundo_nombre: VARCHAR(50), Opcional. Segundo nombre del personal.
  • tercer_nombre: VARCHAR(50), Opcional. Tercer nombre del personal.
  • rol: VARCHAR(50), Obligatorio. Rol del personal (ej. 'Docente', 'Secretaria', 'Administrador').
  • direccion: Referencia a Entidad Dirección, Obligatorio. Dirección del personal.
  • telefono: VARCHAR(20), Obligatorio. Número de teléfono de contacto.
  • email: VARCHAR(100), Opcional. Correo electrónico de contacto.
  • sala_id: Referencia a Entidad Sala, Opcional. Sala principal a la que está asignado el personal.

Entidad: Dirección

  • calle: VARCHAR(100), Obligatorio. Nombre de la calle.
  • numero: VARCHAR(10), Obligatorio. Número de la dirección.
  • ciudad: VARCHAR(50), Obligatorio. Ciudad.
  • provincia: VARCHAR(50), Obligatorio. Provincia.
  • codigo_postal: VARCHAR(10), Opcional. Código postal.

Entidad: ContactoEmergencia

  • nombre_completo: VARCHAR(100), Obligatorio. Nombre completo del contacto de emergencia.
  • relacion: VARCHAR(50), Obligatorio. Relación con el alumno.
  • telefono: VARCHAR(20), Obligatorio. Número de teléfono de contacto de emergencia.

Entidad: Sala

  • nombre: VARCHAR(50), Obligatorio. Nombre de la sala (ej. 'Sala Roja', 'Sala Verde').
  • capacidad: INT, Obligatorio. Capacidad máxima de alumnos.

Entidad: Asistencia

  • alumno_id: Referencia a Entidad Alumno, Obligatorio. Alumno al que corresponde la asistencia.
  • fecha: DATE, Obligatorio. Fecha de la asistencia.
  • estado: ENUM('Presente', 'Ausente', 'Tardanza', 'Justificado'), Obligatorio. Estado de la asistencia.
  • tipo_licencia: VARCHAR(50), Opcional. Tipo de licencia si aplica (ej. 'Enfermedad', 'Personal').

Entidad: Calendario

  • fecha: DATE, Obligatorio. Fecha del evento.
  • descripcion: TEXT, Obligatorio. Descripción del evento.
  • tipo_evento: ENUM('Feriado', 'Licencia', 'Actividad Jardin', 'Actividad Sala', 'Actividad Comunidad', 'Actividad Pais'), Obligatorio. Tipo de evento.
  • sala_id: Referencia a Entidad Sala, Opcional. Sala asociada al evento (si es una actividad de sala).

Entidad: Actividad

  • nombre: VARCHAR(100), Obligatorio. Nombre de la actividad (ej. 'Música', 'Expresión Corporal').
  • descripcion: TEXT, Opcional. Descripción de la actividad.
  • horario: VARCHAR(50), Opcional. Horario de la actividad.
  • docente_id: Referencia a Entidad Personal, Opcional. Docente a cargo de la actividad.

Entidad: Conversacion

  • id: INT, Clave Primaria. Identificador único de la conversación.
  • sala_id: Referencia a Entidad Sala, Opcional. Sala a la que está asociada la conversación (si aplica).
  • fecha_creacion: DATETIME, Obligatorio. Fecha y hora de creación de la conversación.

Entidad: ParticipanteConversacion

  • id: INT, Clave Primaria. Identificador único del participante en la conversación.
  • conversacion_id: Referencia a Entidad Conversacion, Obligatorio. Conversación a la que pertenece el participante.
  • padre_tutor_id: Referencia a Entidad Padre/Tutor, Opcional. ID del padre/tutor participante.
  • personal_id: Referencia a Entidad Personal, Opcional. ID del personal participante.
  • tipo_participante: ENUM('Padre/Tutor', 'Personal'), Obligatorio. Tipo de usuario participante.

Entidad: Mensaje

  • id: INT, Clave Primaria. Identificador único del mensaje.
  • conversacion_id: Referencia a Entidad Conversacion, Obligatorio. Conversación a la que pertenece el mensaje.
  • remitente_padre_tutor_id: Referencia a Entidad Padre/Tutor, Opcional. ID del padre/tutor que envió el mensaje.
  • remitente_personal_id: Referencia a Entidad Personal, Opcional. ID del personal que envió el mensaje.
  • contenido: TEXT, Obligatorio. Contenido del mensaje.
  • fecha_envio: DATETIME, Obligatorio. Fecha y hora de envío del mensaje.
  • leido: BOOLEAN, Obligatorio. Indica si el mensaje ha sido leído.
