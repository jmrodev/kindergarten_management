Estrategia de Pruebas

Se debe implementar una estrategia de pruebas integral para asegurar la calidad y el correcto funcionamiento del sistema.

  • Pruebas Unitarias: Para verificar el correcto funcionamiento de los componentes individuales del código.
  • Pruebas de Integración: Para asegurar que los diferentes módulos del sistema interactúan correctamente entre sí.
  • Pruebas de Sistema: Para validar el sistema completo contra los requisitos funcionales y no funcionales.
  • Pruebas de Aceptación de Usuario (UAT): Realizadas por los usuarios finales (personal del jardín de infantes) para asegurar que el sistema cumple con sus expectativas y necesidades.

Desarrollo Guiado por Pruebas (TDD)

Considerar la adopción de TDD, donde las pruebas se escriben antes que el código de producción. Esto ayuda a asegurar que cada funcionalidad esté cubierta por pruebas y mejora el diseño del código.

Herramientas de Pruebas

Selección de herramientas adecuadas para cada tipo de prueba.

  • Unitarias/Integración: JUnit (Java), Pytest (Python), Jest (JavaScript), PHPUnit (PHP).
  • API (End-to-End): Postman, Insomnia, Hurl (como se ve en el proyecto actual).
  • UI/E2E: Selenium, Cypress, Playwright.

Ejemplos de Pruebas con Hurl (para API)

Siguiendo el ejemplo del proyecto actual, se pueden definir archivos .hurl para probar los endpoints de la API.

Entorno de Pruebas

Configuración de un entorno de pruebas aislado que replique el entorno de producción lo más fielmente posible, pero con datos de prueba.

Consideraciones:

  • Base de datos de pruebas separada.
  • Datos de prueba anonimizados o generados.
  • Automatización del despliegue en el entorno de pruebas.
